# Project Map

This document is the quick orientation layer for agents and collaborators.

## Top-Level Areas

- `frontend/` - browser-facing code and static assets
- `api/` - server routes, handlers, and shared contracts
- `jobs/` - background tasks, maintenance scripts, and batch workflows
- `infra/` or `ansible/` - deployment and environment automation
- `docs/` - planning notes, implementation notes, and operator guidance

## How To Use This Folder

Read planning docs first when you need scope, requirements, or product intent.
Read implementation docs next when you need architecture, operational expectations, or integration boundaries.

## Recommended Read Order

1. `docs/planning/core.md`
2. The most relevant planning doc for the area you are touching
3. The matching file in `docs/implementation/`

## Notes

- Keep this map high level.
- Do not treat this file as the source of truth for requirements.
- Add links here only when they help someone find the right doc faster.
