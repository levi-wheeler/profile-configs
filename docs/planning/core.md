# Planning: Core

## Purpose

Define the durable goals, constraints, and non-goals for the project.

## Product Shape

This repository represents a content-heavy web application with:

- a public-facing experience
- an authenticated administrative surface
- background jobs or scheduled processing
- deployment automation

## Core Goals

- Keep the public experience fast and easy to understand.
- Make administrative workflows clear and operator-friendly.
- Prefer predictable automation over fragile manual procedures.
- Keep requirements and implementation notes separate.

## Non-Goals

- Capturing every historical decision in one file
- Describing internal implementation line by line
- Treating temporary experiments as permanent product scope

## Working Rules

- Planning docs define scope.
- Implementation docs describe how the current system works.
- Non-trivial product changes should be reflected here before or alongside implementation work.
