# Planning: Frontend

## Purpose

Describe the intended shape of user-facing and operator-facing browser experiences.

## Public Surface Goals

- Fast initial load
- Clear navigation
- Strong fallback behavior when live data is unavailable
- Shareable, durable URLs for important views

## Admin Surface Goals

- Task-oriented workflows
- Clear status, progress, and recovery states
- Safe controls for actions with operational impact

## General Frontend Expectations

- Static-first where practical
- Progressive enhancement over hard dependency on complex client state
- Accessible inputs, navigation, and status messaging

## Out Of Scope For This File

- Detailed API payloads
- Per-component implementation notes
- Deployment or runtime internals
