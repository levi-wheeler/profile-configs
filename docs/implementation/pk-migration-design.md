# Design Note: Example Data Migration

## Purpose

This file is a placeholder for design-only migration notes.

## Recommended Structure

- problem statement
- current constraint or pain point
- proposed migration shape
- rollout order
- rollback strategy
- risks and open questions

## Guidance

- Keep design notes implementation-oriented.
- Do not treat them as approved execution plans by default.
- Move reusable guidance into a template if this pattern appears often.
